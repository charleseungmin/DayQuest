# crashlytics-triage

## 언제 사용하는지
- Crashlytics 신규/급증 이슈가 발생했을 때 사용한다.

## 입력 조건
- 이벤트 시그니처
- 영향 사용자/버전/디바이스 분포
- 재현 정보 또는 관련 로그

## 실행 절차
1. 이벤트를 signature 기준으로 묶고 빈도를 계산한다.
2. 심각도와 사용자 영향도로 우선순위를 산정한다.
3. 재현 가능성을 평가하고 가설을 세운다.
4. 수정 후보와 검증 방법을 정의한다.
5. 결과를 구현 계획 또는 blocker 분석에 반영한다.

## 실패 시 처리
- 재현 불가면 로그 보강 액션을 정의한다.
- 데이터 부족이면 추가 수집 항목을 지정한다.

## 출력 형식
- triage 결과 요약
- 우선순위
- 권장 액션
- 검증 계획
